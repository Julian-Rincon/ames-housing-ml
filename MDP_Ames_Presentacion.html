Contexto del problema

¿Por qué un MDP para valuación?

El precio no basta — el agente debe decidir qué hacer con él

01 · El problema real
Un tasador humano evalúa una propiedad y comete errores del 15–25% en segmentos premium. Un error del 10% sobre $300K equivale a $30,000 de diferencia.

Necesitamos un agente autónomo que decida: ¿confío en la predicción del modelo? ¿La envío a revisión humana? ¿Rechazo la solicitud por datos insuficientes?
02 · Nuestro modelo base: K-Means + XGBoost
🔵 20,203 propiedades de Ames, Iowa (2006–2024)
🟢 R²=0.9609 — modelo híbrido K-Means (k=6) + XGBoost
🟡 MAE=$26,752 — error promedio de valuación
🔵 6 clusters de mercado como espacio de estados
El modelo predice el precio. El MDP decide la acción sobre esa predicción.
03 · El MDP: decisiones secuenciales con memoria
El ML predice un número. El MDP toma decisiones secuenciales considerando el costo de equivocarse hoy sobre el valor futuro del portafolio.
# Ecuación de Bellman — el corazón del agente
V*(s) = maxa [
  R(s,a) + γ · Σs' P(s'|s,a) · V*(s')
]

reward_function.py · Función de recompensa económica
# Recompensa basada en impacto económico real # Error del modelo → costo financiero para el banco/propietario
def calc_reward(pred_price, real_price, action):
    error = abs(pred_price - real_price) / real_price
    if action == 'APROBAR':
        if error < 0.10: return +200
        elif error < 0.25: return -500
        else: return -2000 # error > 25%: crítico
    elif action == 'REVISAR':
        return -150 if not corrected else -50
    elif action == 'RECHAZAR':
        return +50 if insufficient else -200

Parte 2 del taller · Value Iteration

Iteración de valor: convergencia real

γ=0.95 · θ=0.0001 · Convergió en 259 iteraciones sobre 6 estados

value_iteration.py · Implementación completa
# Parámetros del agente SAVI GAMMA = 0.95  # Factor de descuento THETA = 0.0001 # Umbral convergencia
V = {s: 0 for s in STATES} # V(s) inicial = 0
while True:   delta = 0   V_new = V.copy()
  for s in STATES:     q_sa = {}     for a in ACTIONS: # APROBAR, REVISAR, RECHAZAR       reward = R_dict[s][a]
      # Valor futuro esperado       efv = sum(P_dict[s][ns] * V[ns]               for ns in P_dict[s])
      # Bellman       q_sa[a] = reward + GAMMA * efv
    # Maximizar Q(s,a)     V_new[s] = max(q_sa.values())     delta = max(delta, abs(V[s] - V_new[s]))
  V = V_new   if delta < THETA: break
# Convergió en 259 iteraciones ✓
V*(s) — Valor óptimo por estado
Parte 3 del taller · Política óptima π*

La política que aprendió el agente

Lo que emerge de la ecuación de Bellman — no fue programado, fue aprendido

Distribución de la política óptima π* por estado
Las 3 reglas de oro — análisis profundo
REGLA 01
S4 y S5 siempre APRUEBAN
Los clusters más grandes (31.1% y 32.5% del dataset) tienen el menor error del modelo. R(APROBAR) llega a ≈ −$7.4, mejor que R(REVISAR) ≈ −$127. El agente prefiere APROBAR porque la pérdida esperada es menor que el costo de revisión.
REGLA 02
S0 y S2 siempre REVISAN
Clusters con alta variabilidad interna. El agente aprendió que aquí el costo de APROBAR incorrectamente (−$500 a −$2,000) supera el de REVISAR (−$150). Comportamiento emergente, no programado.
REGLA 03 — LA MÁS INTERESANTE
S3 RECHAZA (solo 7 casos, 0.03%)
El cluster más pequeño. R(APROBAR)=−$2,000, R(RECHAZAR)=+$50. El agente descubrió que pedir más datos vale más que valuar con incertidumbre alta. Refleja perfecto manejo del riesgo.
